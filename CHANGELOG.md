# Changelog

All notable changes to this project are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added

- **URP/Lit transparent material path** (`Assets/Demo/Runtime/MaterialFactory.cs::ConfigureFractureTransparent`) — fragments now use stock URP/Lit configured at runtime (`_Surface=1`, `_Cull=0`, `_ZWrite=1`, `_SURFACE_TYPE_TRANSPARENT` keyword + render queue + tag). The standalone `MeshFractureDemo.cs` example uses the same recipe inline for its default interior material. Fade animates `_BaseColor.a` 1 → 0 directly on the material instance — `MaterialPropertyBlock` overrides on URP/SRP-Batcher shaders are silently dropped on some GPU/driver/keyword combos, which presented as fragments popping out instantly when their lifetime expired. Direct material mutation always reaches the GPU.
- **Optional Unity physics mode.** `FractureBurst.UseUnityPhysics = true` gives each fragment a `Rigidbody` + convex `MeshCollider` (or `BoxCollider` fallback for fragments past the convex-hull vert ceiling) so chunks bounce off any collider in the scene. Tunable per-burst via `Bounciness`, `Friction`, `FragmentMass`. Pair-wise `Physics.IgnoreCollision` between siblings runs once after the build completes — without it, the depenetration impulse on shared cell boundaries launches half the burst skyward at spawn.
- **Pre-cooked convex collider hulls.** `FragmentCache.CachedData.ColliderMeshes` paired 1:1 with `Meshes`. `FragmentCache.BuildAndBakeColliderMesh` position-dedups the visual fragment mesh (the cap-flat-normal duplication doubles the input to Unity's hull algorithm without contributing unique geometry — strip it before cooking) and runs `Physics.BakeMesh(convex: true)` on the result during the worker frame. Consumers assign `MeshCollider.sharedMesh = cached.ColliderMeshes[i]` and Unity reuses the cached PhysX shape — that's the difference between a 5–10 ms hitch per fragment and zero. Uses Unity 6's new `Mesh.GetEntityId()` overload (`int instanceID` is deprecated). Skipped automatically when the deduped vert count exceeds 100, since Unity's convex hull tops out at 256 polys (≈ 130 input verts) and FractureBurst falls back to BoxCollider for those fragments anyway.
- **Progressive fragment build** in `FractureBurst`. `Initialize` now only captures inputs and pre-computes per-fragment velocities + tumble; `Update` then instantiates `BUILD_BATCH_SIZE = 2` fragments per tick. The previous monolithic Initialize cost ~50–180 ms for an 8-fragment burst with full physics + trails (Rigidbody + MeshCollider + TrailRenderer + Material allocation × 8) and produced visible jitter on Fracture-All. New `IsFullyInitialized` property lets callers serialize bursts (DemoBootstrap waits for each burst's build to finish before starting the next).
- **Settle-based dissolve.** `DissolveAfterSettle = true` waits for every fragment to come to rest (linear velocity² < 0.04, angular velocity² < 0.25) for `SettleHoldDuration` before fading. Time-based fallback at `Lifetime − DissolveDuration` always fires regardless of motion — so fragments that bounce forever, slide off the stage, or otherwise never trip the settle thresholds still fade rather than pop. Hard-destroy at `2× Lifetime` as a final safety net.
- **Watertightness fixes** in `MeshFragmenter`. Cap polygons from earlier clip planes now get re-clipped + re-tagged when later planes cross them; cut-edge intersections dedup by world position (`DedupCutEdgesByPosition`) before chaining, so the cut-edge graph stays connected where two clip planes meet inside the cell; per-plane `LerpCache` reuses the same intersection vertex when two triangles share a cut edge; `ChainEdgesAll` walks the graph and returns ALL closed loops (not just the first) so non-convex source meshes whose boundary disjoints into multiple regions get every region capped. The previous single-loop / non-deduped implementation produced "see-through to interior" cells where two cuts met.
- **Solidify-for-fragment** for hollow geometry (`DemoTargetCatalog.Definition.SolidifyForFragment`). Open-topped kenney crates and chests have inner + outer wall triangles but no volume between them; Voronoi cells on the raw mesh produced paper-thin shell pieces that read as fractured paper. The merge path keeps the carved exterior silhouette AND adds an interior filler box at 0.85× extents (UVs set to the source's average UV so it samples the same atlas swatch as the carved walls) so cells span both the carved walls and solid wood mass — fragments come out chunky without losing the crate's outer shape. Pre-fracture visual still uses the original mesh; only the fragmentation source is the merged mesh.
- **WebGL demo scene** at `Assets/Demo/Scenes/MeshFractureDemo.unity` — eight pedestals on a dynamically-sized circle (cube, sphere, five Kenney pirate-kit props, one Kenney character), click-to-fracture, drag-to-orbit, pinch + wheel zoom, fragment-count slider (2–24), explosion-force slider, trails toggle, fracture-all / restore-all buttons, hotkey legend (Space / R / 1 / 2 / 3). Showcase wall placed behind one target slot so chunks bounce off a real collider.
- **`Assets/Demo/Runtime/`** — `DemoBootstrap.cs` (scene + camera + targets + UI), `DemoTargetCatalog.cs` (8 fracturable definitions), `DemoTarget.cs` (per-target controller + solidify-merge helpers), `MaterialFactory.cs` (URP/Lit + transparent setup + kenney atlas binding), `DemoUI.cs` (UI Toolkit overlay, ds-* design system styled, mobile breakpoint flip).
- **`Assets/Demo/Runtime/UI/DemoUI.uxml`** — UI Toolkit overlay layout consuming the [`unity-ui-document-design-system`](https://github.com/sinanata/unity-ui-document-design-system) submodule (vendored at `Vendor/unity-ui-document-design-system`, junctioned into `Assets/DesignSystem`). Title block with credits row, status toast, controls panel, instructions / hotkeys panel, GitHub promo button.
- **`Assets/Demo/Editor/DemoModelImporter.cs`** — `AssetPostprocessor` that flips Read/Write on imported FBXes (the fracturer needs `mesh.vertices` access at runtime), strips the auto-generated colliders, sets `materialImportMode = None` so the kenney FBXes don't drop orphan placeholder materials, and configures the colormap atlas (no mipmaps, clamp wrap, bilinear, max 512).
- **`Assets/WebGLTemplates/MeshFractureDemo/index.html`** — HiDPI-aware WebGL loader with branded splash, mobile viewport meta, Apple-style status bar, Brotli decompression fallback. `matchWebGLToCanvasSize: true` for crisp rendering on Retina; `WebGLDevicePixelRatio.jslib` bridges `window.devicePixelRatio` to Unity so `PanelSettings.scale = DPR` keeps a 36 px UI button rendering at 36 CSS-px on every device.
- **`Tools/Build/Build-Demo.ps1`** — Windows orchestrator for the WebGL build. Single PowerShell command builds, optionally serves on `:3000` for smoke-test, optionally force-pushes to `gh-pages` for [the live preview](https://sinanata.github.io/unity-mesh-fracture/). Mirrors the unity-ui-document-design-system orchestrator's recovery logic (Burst-AOT cache retry, native NTSTATUS crash labels, process-tree kill on Ctrl+C, stale `Temp/UnityLockfile` removal, JSON build report, live phase progress from Unity's `DisplayProgressbar:` log markers).
- **`Tools/Build/Deploy-GhPages.ps1`** — single-commit force-push to the `gh-pages` orphan branch via `git worktree`. The branch always contains exactly one commit, so the public repo doesn't accumulate ~10 MB build artefacts per deploy.
- **`Assets/Editor/BuildCli.cs`** — Unity batchmode entry point (`MeshFractureDemo.BuildTools.BuildCli.BuildWebGL`). Re-asserts `compressionFormat = Brotli`, `decompressionFallback = true`, `template = "PROJECT:MeshFractureDemo"` defensively at build time. Writes a JSON report to `Tools/Build/output/report-*.json` so the orchestrator validates success without scraping the log.
- **Project skeleton** — `Packages/manifest.json` (URP 17.3.0, Input System 1.18, AI Navigation, Visual Scripting), `ProjectSettings/`, default URP global settings + volume profile.
- **Tap-vs-drag gesture arbitration** in `DemoBootstrap`. Primary-pointer gestures start as candidate-tap and promote to drag once total movement exceeds a DPR-scaled threshold (`DRAG_THRESHOLD_CSSPX = 11`). Same code path handles mouse + touch — no platform branches. Right-click drag remains a desktop power-user shortcut for orbit without tap-arbitration.
- **Pinch-to-zoom** (touch) and **wheel zoom** (desktop) with platform-aware sensitivity. Wheel handling normalizes Windows mouse-wheel notches (~120/notch) and trackpad smooth-scroll (~1–10/event) to the same effective range, then clamps browser momentum spikes (~300+) to a saturation point. Both write to a target distance and `SmoothDamp` eases the live distance.
- **HiDPI panel scale** (`DemoUI.MakePanelSettings`). `panelSettings.scale = devicePixelRatio` keeps UI components rendering at their declared CSS-pixel size on Retina / mobile devices. Without this a 36 px button rendered at 18 CSS-px on a 5K iMac and 12 CSS-px on an iPhone 14 Pro Max because Unity's `Screen.width` reports buffer pixels. Mirrors the fix shipped in `unity-ui-document-design-system` 1.2.0.
- **Status toast + Baking-state UI**. The Fracture-All button doubles as the loading indicator: while pre-bakes are pending, it disables, the refresh icon spins (driven by `DesignSystemRuntime`), and the label flips to "Baking...". Slider value labels surface in the section header (the design system's showcase pattern), not via show-input-field which inherits the editor-theme inset box.
- **`AreAllBakesReady` gating** for tap / Space / FractureAll. The pre-bake worker yields one bake per frame; an 8-target circle finishes in ~10 frames after scene load. Triggering FractureAll before all bakes complete previously caused eight back-to-back synchronous BakeSync calls (~0.5 s freeze); now the coroutine waits and stages bursts one per frame.

### Removed

- **Legacy custom URP shader** (and the now-empty `Assets/MeshFracture/Resources/` tree). The runtime no longer ships any `.shader` assets — fragment materials are stock URP/Lit configured at runtime via the 10-line `ConfigureFractureTransparent` helper. The custom shader path was the source of a long-running WebGL2 cross-compilation bug (fragments rendered solid black on Chrome/WebGL2 only, despite three rounds of speculative fixes); stock URP/Lit sidesteps the cross-compiler entirely and removes ~220 lines of HLSL plus its `EnsureBuildShaders` AlwaysIncludedShaders entry. The single-script `MeshFractureDemo.cs` example now configures URP/Lit transparent inline for its default interior material.

### Changed

- **README.md, CHANGELOG.md, CONTRIBUTING.md** comprehensively updated to match the shipped runtime. Documented the URP/Lit transparent path, added the `UseUnityPhysics` / `Bounciness` / `Friction` / `FragmentMass` / `DissolveAfterSettle` / `SettleHoldDuration` / `PreBakedColliderMeshes` / `IsFullyInitialized` API surface, added `CachedData.ColliderMeshes` to the cache shape, refreshed the architecture, what-makes-this-robust, and when-not-to-use sections.

### Fixed

- **Fragments popped instantly at lifetime end** instead of fading. The hard `Destroy(gameObject)` at `Lifetime` ran before the dissolve trigger; settle-based bursts whose fragments never settled hit Lifetime with `dissolving = false` and got destroyed without playing the fade. Fix: the dissolve trigger now ORs settle-based and time-based readiness — whichever fires first wins. Hard-destroy moves to `2× Lifetime` as a defensive safety net.
- **Carved kenney crate / chest fragmented as paper-thin shells.** The imported FBXes have inner and outer wall triangles but no volume between them; Voronoi cells produced fragments with see-through interiors. Fix: `DemoTargetCatalog.Definition.SolidifyForFragment` opt-in. When set, `DemoTarget.BuildSolidifiedMergedMesh` merges the carved exterior with an interior filler box at 0.85× extents (UVs averaged from the source so the filler samples the same atlas swatch); the fragmenter sees the union, cells span both walls and solid wood, fragments come out chunky while preserving the carved silhouette.
- **`MaterialPropertyBlock` alpha override silently dropped on URP+WebGL.** SRP-Batcher with the legacy custom shader sometimes failed to honour PB-overridden properties declared inside `UnityPerMaterial` CBUFFER on certain GPU/driver/keyword combos — the alpha never reached the shader and fragments stayed opaque to lifetime end. Fix: `FractureBurst` now mutates the burst's own material instances directly (`SetColor("_BaseColor", ...)` with `c.a = alpha`), which always reaches the GPU. Caller hands fresh material instances per burst (DemoTarget creates copies on every Fracture()), so direct mutation is safe.
- **`FractureBurst` was using deprecated `BakeMesh` overload** in Unity 6. Switched `FragmentCache.BuildAndBakeColliderMesh` to `mesh.GetEntityId()` instead of the int-instanceID overload (CS0618 deprecation warning).

## [0.1.0] — 2026-05-02

Initial cut. Extracted from [Leap of Legends](https://leapoflegends.com)' destruction system after one year of weekly use.

### Added

- `MeshFragmenter.Fragment(mesh, count, center)` — pure-C# Voronoi fracturer using iterative half-space clipping. Returns `FragmentResult[]`; each fragment has 2 submeshes (exterior + cap) so cuts can carry a separate "raw interior" material. Cap faces get their own duplicated vertices with the cap-plane normal so lighting on the cut face stays flat instead of lerping into the original surface normals.
- `MeshFragmenter.BakeSkinnedMesh(smr)` — utility to capture a skinned mesh's current animated pose as a static `Mesh`, suitable for fracturing. Uses the explicit `BakeMesh(mesh, useScale: false)` overload so callers that pre-scale by `lossyScale` don't get the scale applied twice.
- `FragmentCache.RequestPreBake(key, prefab, count)` — async pre-bake worker. Reads the prefab's bind-pose `sharedMesh` immediately (via the first `SkinnedMeshRenderer` child, then falls back to `MeshFilter`), defers Voronoi fragmentation to the next frame so the visual creation frame stays light. Coroutine-driven; multiple requests don't stack into one frame. Idempotent — duplicate keys are ignored.
- `FragmentCache.BakeSynchronous(key, mesh, rot, count)` — synchronous bake from an arbitrary mesh. Use during loading screens.
- `FragmentCache.TryGet`, `Evict`, `Clear` — lookup and lifecycle.
- `FractureBurst` MonoBehaviour — per-fragment lifecycle. Spawns one GameObject per cached fragment, simulates ballistic motion + tumble + ground bounce on the CPU with a cheap `y = -1` floor, drives a noise-modulated `TrailRenderer`, fades the alpha, self-destructs at lifetime end.
- `FractureBurst.LockToXY` — flag for side-on 2D games. Locks fragment Z and tumble axes to the XY plane.
- `FractureBurst.EnableTrails` — false to skip TrailRenderer setup entirely (mobile perf knob).
- `FractureBurst.GravityVector` — non-Y gravity vector (e.g. low-grav arenas, sideways gravity).
- `MeshFractureDemo` — Space-to-shatter demo. Drop on a Cube, press Play, press Space. Configures URP/Lit transparent (`_Surface=1`, `_Cull=0`, `_ZWrite=1`) at runtime for its default interior material.
